# admin-next-dev 第三阶段人工验收脚本

适用地址：

- 后台测试入口：`http://127.0.0.1:807/admin-next-dev/`
- 数据中心页：`http://127.0.0.1:807/admin-next-dev/#/analytics`
- 导出中心页：`http://127.0.0.1:807/admin-next-dev/#/exports`

测试前准备：

- 使用一个已有后台权限的账号登录。
- 准备一组能查出真实数据的商家、日期范围、商品分类。
- 浏览器建议优先用 Chrome。
- 打开浏览器开发者工具，便于观察接口报错和移动端视图。

## 一、基础访问验收

### 1. 页面可访问

步骤：

1. 打开 `http://127.0.0.1:807/admin-next-dev/`
2. 登录后台
3. 依次进入 `#/analytics` 与 `#/exports`

预期结果：

- 页面能正常打开，不白屏。
- 顶部标题、筛选区、摘要区、主体卡片都能正常渲染。
- 没有明显样式错乱、按钮重叠、控制台红色报错。

异常点记录：

- 页面空白
- 卡片样式错位
- 筛选组件不显示
- 控制台存在未捕获异常

### 2. 路由可直达

步骤：

1. 直接访问 `#/analytics`
2. 直接访问 `#/exports`
3. 刷新浏览器

预期结果：

- 页面刷新后仍停留在当前模块。
- 当前页 query 参数不会丢失。

异常点记录：

- 刷新后跳回首页
- 刷新后 query 丢失
- 刷新后重新登录且无法回到当前页

## 二、/analytics 人工验收

### 1. 默认态验收

步骤：

1. 打开 `#/analytics`
2. 不修改任何条件，等待页面首次加载完成

预期结果：

- 默认展示“近 7 天 / 全部商家”一类的默认条件。
- 顶部有当前操作人、实时后端数据、刷新数据按钮。
- 摘要横条显示统计范围、统计粒度、预警数量、当前操作人。
- 有四块趋势图区域。
- 榜单区包含 Top 卡片和下方表格。
- 预警区包含预警强化卡片和下方表格。

异常点记录：

- 默认条件和摘要区不一致
- 预警数量与下方列表数量不一致
- Top 卡片有内容但下方表格为空

### 2. 快捷时间切换

步骤：

1. 依次点击快捷时间按钮
2. 观察 query 是否变化
3. 观察图表和榜单是否刷新

预期结果：

- 被点击的快捷时间按钮高亮。
- URL 中 query 同步变化。
- 图表、榜单、预警数据跟随刷新。
- 若无数据，图表显示“暂无趋势数据”，表格显示空态。

异常点记录：

- 按钮高亮不切换
- URL 未更新
- 数据未刷新
- 图表报错或空图不显示提示

### 3. 日期范围 / 月份 / 粒度联动

步骤：

1. 选择一个日期范围并点击“应用筛选”
2. 再切换为月份筛选并点击“应用筛选”
3. 切换统计粒度为“按天”或“按月”

预期结果：

- 选择日期范围后，快捷时间应被清空。
- 选择月份后，日期范围应被清空。
- 当前分析条件标签区能准确反映筛选值。
- 摘要横条中的统计范围、粒度与筛选保持一致。

异常点记录：

- 日期范围和月份同时生效
- 条件标签与实际筛选不一致
- 粒度变化后图表维度未变化

### 4. 多条件组合筛选

步骤：

1. 选择商家
2. 选择商家状态、到期状态、商品分类
3. 填写金额区间
4. 点击“应用筛选”

预期结果：

- 条件标签区完整展示已选条件。
- 榜单、图表、预警都按新条件刷新。
- 重置后恢复默认条件。

异常点记录：

- 某个条件没有进入 URL
- 重置后仍残留旧条件
- 金额区间输入后接口异常

### 5. 榜单强化卡片验收

步骤：

1. 查看热销商品 Top3 卡片
2. 查看商家成交 Top3 卡片
3. 点击商家 Top3 卡片

预期结果：

- 商品卡片展示 Top 排名、标题、销量/订单、成交额。
- 商家卡片展示 Top 排名、商家名、审核/到期状态、成交额。
- 点击商家卡片后打开商家分析抽屉。

异常点记录：

- Top 卡片和表格数据不一致
- 点击卡片无响应
- 抽屉打开但数据为空

### 6. 预警强化卡片验收

步骤：

1. 观察预警区顶部强化卡片
2. 点击“查看商家”

预期结果：

- 高风险、需跟进、提示类卡片颜色区分清晰。
- 预警卡片文案、商家、指标值能正常展示。
- 点击“查看商家”后抽屉打开并加载对应商家数据。

异常点记录：

- 级别颜色不对
- 预警卡片内容和表格不一致
- 点击后加载错商家

### 7. 商家抽屉验收

步骤：

1. 从商家排行点击商家名
2. 从预警卡片点击“查看商家”
3. 从预警表格点击“查看”

预期结果：

- 三种入口都能打开抽屉。
- 抽屉展示商家基础信息和概览指标。
- 切换不同商家时内容随之更新。

异常点记录：

- 抽屉复用旧数据
- loading 一直不消失
- 某个入口打开的是错误商家

### 8. 前进 / 后退验收

步骤：

1. 在 `#/analytics` 依次切换两到三次不同筛选
2. 点击浏览器“后退”
3. 再点击浏览器“前进”

预期结果：

- URL query 会回到上一组条件。
- 页面筛选控件会回填。
- 图表、榜单、预警会自动按回退后的条件重载。

异常点记录：

- URL 回退了，但页面筛选没变
- 页面筛选变了，但数据没刷新
- 后退 / 前进触发重复请求或页面闪烁明显

## 三、/exports 人工验收

### 1. 默认态验收

步骤：

1. 打开 `#/exports`
2. 等待页面首次加载完成

预期结果：

- 顶部有导出中心标题、当前操作人、刷新预览按钮。
- 摘要横条显示预览范围、预警数量、最近导出、当前操作人。
- 导出模块卡片完整显示。
- 下方有热销商品预览和最近导出记录。

异常点记录：

- 卡片数量不对
- 最近导出为空但仍显示脏数据
- 预览区和摘要区范围不一致

### 2. 条件筛选验收

步骤：

1. 选择快捷时间
2. 再切换日期范围
3. 选择商家、商品分类、订单状态
4. 点击“应用筛选”

预期结果：

- URL query 与当前条件同步。
- 当前导出条件标签区准确展示筛选。
- 预览卡片和热销商品预览随条件变化。

异常点记录：

- URL 与筛选不一致
- 标签未展示某些条件
- 预览数据未刷新

### 3. 模块权限态验收

步骤：

1. 观察四个导出模块卡片
2. 对比当前账号应有权限范围
3. 分别点击可用模块和禁用模块

预期结果：

- 有权限模块显示对应 CSV 标识和“立即导出”按钮。
- 无权限模块显示“无权限”或“需后台授权”，按钮禁用。
- 点击无权限模块不会发起导出，只提示权限不足。

异常点记录：

- 无权限模块仍能点击导出
- 有权限模块被错误禁用
- 权限标识与实际行为不一致

### 4. 实际导出验收

步骤：

1. 分别点击订单导出、商家导出、续费记录导出、平台统计导出
2. 检查浏览器是否下载 CSV
3. 检查文件名是否沿用后端返回名

预期结果：

- 点击后出现“导出已开始”提示。
- 有权限模块能触发下载。
- 文件名优先使用后端返回文件名。
- 最近导出记录新增一条。

异常点记录：

- 点击后无下载
- 文件名固定死值
- 最近导出记录未新增
- 同一模块重复点击卡住在“导出中”

### 5. 导出记录验收

步骤：

1. 连续发起两到三次导出
2. 刷新当前页面
3. 点击“清空记录”

预期结果：

- 最近导出记录保留在当前浏览器会话内。
- 刷新页面后记录仍在。
- 清空记录后列表为空。

异常点记录：

- 刷新后记录丢失
- 清空记录无效
- 记录顺序不是最新在最前

### 6. 前进 / 后退验收

步骤：

1. 在 `#/exports` 中切换两到三组不同筛选
2. 点击浏览器“后退”
3. 再点击浏览器“前进”

预期结果：

- 页面筛选控件会跟随 query 恢复。
- 预览卡片和榜单会自动刷新。

异常点记录：

- query 恢复但界面未恢复
- 界面恢复但数据未刷新

## 四、移动端验收

建议视口：

- `390 x 844`
- `768 x 1024`

### 1. /analytics 移动端

步骤：

1. 打开开发者工具切到手机视图
2. 访问 `#/analytics`
3. 滚动查看筛选区、摘要区、图表区、榜单区、预警区

预期结果：

- 摘要横条单列堆叠。
- 趋势图、榜单卡片、预警卡片转为单列。
- 按钮和标签不重叠。
- 抽屉在移动端仍可正常打开。

异常点记录：

- 筛选组件溢出
- 卡片文字被截断
- 图表宽度超出容器
- 抽屉内容横向滚动严重

### 2. /exports 移动端

步骤：

1. 切到手机视图打开 `#/exports`
2. 滚动查看导出卡片和历史记录

预期结果：

- 导出卡片单列展示。
- 权限态和状态条不会换行错乱。
- 历史记录块可完整显示文件名和时间。

异常点记录：

- 状态条换行后错位
- 导出按钮超出容器
- 文件名显示被压坏

## 五、重点异常清单

本轮验收建议优先盯这几类问题：

- query 已变化，但页面没有按回退条件自动重载
- 权限态误判，导致导出卡片被错误禁用或错误放开
- 榜单强化卡片与下方表格数据不一致
- 预警强化卡片与表格商家不一致
- 导出成功但历史记录未更新
- 移动端下筛选区、状态条、卡片发生挤压错位
