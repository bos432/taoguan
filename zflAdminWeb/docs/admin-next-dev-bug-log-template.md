# admin-next-dev 问题登记表 + 修复优先级模板

适用范围：

- 测试目录：`public/admin-next-dev`
- 主要页面：`#/analytics`、`#/exports`
- 配套验收脚本：`docs/admin-next-dev-phase3-qa.md`

使用方式：

1. 按人工验收脚本逐项测试。
2. 每发现一个问题，就新增一条问题记录。
3. 用“优先级建议”先做初步归类。
4. 直接把这份表发给我，我就能按单逐个修。

## 一、优先级定义

### P0

定义：

- 页面无法打开
- 白屏
- 核心流程完全不可用
- 导出完全失效
- 路由失控导致无法继续测试

处理建议：

- 立即修
- 优先于所有样式问题

### P1

定义：

- 核心功能可进入，但结果错误
- 筛选条件和实际数据不一致
- 回退前进恢复错误
- 点击商家打开错详情
- 权限态明显误判

处理建议：

- 当天优先修
- 优先于视觉微调

### P2

定义：

- 功能可用，但交互或展示明显不符合预期
- 榜单卡片和表格存在轻微不一致
- 移动端布局错位
- 状态文案不准
- 空态、提示态不完整

处理建议：

- 集中收尾修
- 可和其他同类问题一起处理

### P3

定义：

- 纯视觉细节
- 间距、颜色、字号、阴影、文案顺序等微调

处理建议：

- 最后统一修

## 二、问题状态定义

- `new`：新发现，尚未处理
- `triaged`：已确认，已给优先级
- `fixing`：正在修
- `fixed`：已修复，待回归
- `verified`：已回归通过
- `closed`：确认关闭

## 五、常见问题归类建议

### analytics 常见归类

- `筛选联动问题`
  - 日期范围、月份、快捷时间互相覆盖异常
- `数据展示问题`
  - 榜单卡片、表格、摘要数据不一致
- `预警问题`
  - 预警级别颜色错误、预警卡片和表格不一致
- `详情抽屉问题`
  - 打开错商家、抽屉不刷新、loading 不消失
- `路由恢复问题`
  - 回退前进后 query 恢复但数据不恢复
- `移动端布局问题`
  - 卡片堆叠异常、图表溢出、抽屉不好用

### exports 常见归类

- `筛选联动问题`
  - 条件标签不准、预览不刷新
- `权限态问题`
  - 可导出却被禁用、无权限却能导出
- `导出动作问题`
  - 点击无下载、文件名错误、按钮卡住
- `导出记录问题`
  - 记录未新增、刷新后丢失、清空无效
- `路由恢复问题`
  - 回退前进后条件恢复不完整
- `移动端布局问题`
  - 权限状态条错位、按钮溢出、文件名挤压

## 六、修复优先级建议模板

如果你希望我按批次修，可以直接用下面这个格式发我。

```text
[第一批先修]
- BUG-001
- BUG-003
- BUG-005

[第二批再修]
- BUG-002
- BUG-004

[最后统一微调]
- BUG-006
- BUG-007
```

推荐分批方式：

- 第一批：P0、P1
- 第二批：P2
- 第三批：P3

## 八、推荐提交流程

建议你本地测试时按这个顺序来：

1. 先在 `docs/admin-next-dev-phase3-qa.md` 里按脚本走一遍。
2. 发现问题后，在本模板里登记。
3. 先把 `P0/P1` 发给我，我优先修功能正确性。
4. 再把 `P2/P3` 发给我，我统一做细节收尾。

```text

- 操作：进入页面后查看导出卡片
- 实际：订单导出显示无权限，但当前账号实际可导出
- 期望：订单导出卡片应可点击
```
